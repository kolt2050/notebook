# Правила разработки и внесения правок

Данный файл содержит обязательные параметры и принципы, которыми необходимо руководствоваться при работе над проектом.

## Технические параметры тела документа
- **Размер шрифта**: 11
- **Межстрочный интервал**: 1.5
- **Отступы у изображений**: межстрочное расстояние до и после картинки — 0.5
- **Ширина редактора**: На всю ширину окна, но сохраняя «внутреннюю рамку» (padding, border, rounded corners, shadow).

## Общие принципы разработки
- **Язык интерфейса**: Приложение мультиязычное. Все строки интерфейса должны выноситься в `static/js/i18n.js`.
- **Язык документации**: Основная техническая документация ведется на русском языке.
- **Эстетика**: Дизайн должен быть современным, премиальным и динамичным. Используйте CSS-переменные для тем, плавные переходы (transitions) и современную типографику.
- **Чистота кода**: Самодокументированный код, понятные имена переменных и функций на английском языке.
- **Интерактивность**: Поддержка горячих клавиш (Ctrl+Click для ссылок) и умных функций (авто-ссылки, сортировка).
- **Контейнеризация**: Всегда проверять работоспособность приложения внутри Docker после внесения изменений.

## Правила работы с фронтендом
- Использовать Vanilla CSS и Vanilla JS.
- Приоритет визуальному совершенству: избегать стандартных цветов, использовать гармоничные палитры.
- Все текстовые элементы должны иметь атрибуты `data-i18n` для автоматического перевода.
- Иерархия документов управляется через `tree.js` и поддерживается рекурсивно.

## Процесс внесения правок
1. Анализ текущей реализации.
2. Создание плана реализации в `implementation_plan.md`.
3. Выполнение изменений с сохранением обратной совместимости.
4. Верификация через ручное тестирование в браузере.
5. Обновление `README.md` и `task.md`.
